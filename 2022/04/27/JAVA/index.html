抽象类与抽象对象

当我们将共性的方法，抽取到父类之后，发现这个方法在父类中无法给出具体明确。而且这个方法，还是子类必须要有的方法，就可以设计为抽象方法.

注意事项

  1. 抽象类不能实例化
  2. 存在构造方法
  3. 可以存在普通方法
  4. 抽象类的子类 1）要么重写抽象类中所有抽象方法 2）要么是抽象类
  5. 不能同final,private,static一同使用
  6. 可以定义变量与常量

接口interface

是对规则的声明，java中接口更多体现的是对行为的抽象。

特征：1）没有成员变量 2）没有普通方法

接口的使用：

通过implements关键字实现类

实现类（接口的子类）：

​ 1.重写所有抽象方法

​ 2.将实现类变成抽象类

接口的成员特点：

  1. 成员变量：只能定义常量，因为系统默认加入public static final关键字
  2. 成员方法：只能是抽象方法因为系统默认加入public abstract关键
  3. 构造方法：无

类和接口的关系

实现关系，可以单实现，也可以多实现，可以在继承一个类的同时，实现多个接口

接口和接口的关系：

继承关系，可以单继承，也可以多继承

接口新特性

JDK8：

  • 可以定义非抽象方法，但是需要用default关键字修饰，这些方法就是默认方法。

  • 可以用来解决接口升级的问题

  • 可以定义静态方法，只允许通过接口名调用

JDK9：可以定义私有方法

接口中方法有重复逻辑代码时，抽取为私有方法

static

  • 修饰符，可修饰成员变量、成员方法

​ 1.被类的所有对象所共享

​ 2.多了一种调用方式，可以通过类名进行调用

​ 3.随着类的加载而加载，优先于对象存在

  • 在静态方法中，只能访问静态成员，非静态成员通过对象调用

  • static中不允许使用this关键字

final

  • 可以修饰方法，类，变量

  • 被修饰的表示最终形式，不能重写，继承，赋值

    基础数据类型：数据值不可改变

    引用数据类型：地址值不可改变，内容可变

final修饰变量名规范一单词全大写，多个单词字中间用下划线分割

方法重载与重写

方法重载(Overload)：在同一个类中，方法名相同，参数不同，与返回值无关

方法重写(Override)：在子父类中出现方法名，参数，返回值都相同的方法

方法重写注意事项：

  • 父类中私有方法不能重写
  • 子类重写父类方法时，访问权限必须大于等于父类

继承extends

  • 子类可以直接使用父类中的非私有的成员

  • 子父类中，重名成员变量，优先使用子类，使用super可使用父类

  • 子父类中，出现方法名，参数，返回值都相同的方法时，会对父方法进行重写

  • 只支持单继承，不支持多继承，但支持多层继承

继承中父类构造方法的调用

子类构造方法中第一句默认为super();，用来调用父类的无参构造方法

需要使用不同包内的同名类时，一个import导入，另一个通过包名创建对象

多态

多态前提

  1. 有继承/实现关系

  2. 有方法重写

  3. 有父类引用指向子类对象

对象多态

多用于将方法的形参定义为父类类型，这个方法就可以接受到该父类的任意子类对象了

行为多态

同一个方法，具有多种不同表现形式

多态成员访问特点

  1. 成员变量：编译看左边（父类），运行看左边（父类）
  2. 成员方法：编译看左边（父类），运行看右边（子类）

多态中的转型

  1. 向上转型

    从子到父（父类引用指向子类对象）

  2. 向下转型

    从父到子（将父类引用所指向的对象，转交给子类类型）、

代码块

局部代码块

  • 作用：限定变量的生命周期，提早的释放内存

构造代码块

  • 位置：类中方法外的一对大括号

  • 特点：在创建对象，执行构造方法的时候，就会执行构造代码块，优先构造方法执行

  • 作用：将多个构造方法中，重复的代码，抽取到构造代码块中，提高代码的复用性

静态代码块

  • 位置：类外的一对大括号，需要加入static关键字

  • 特点：随着类的加载而执行，类只加载一次，所以就执行一次

  • 作用：对数据进行初始化；加载配置文件
